板级资源总览

存储与时钟
  • 8 MB OPI PSRAM（来自 SF32LB525UC6），Zephyr 映射到 0x60000000，走 mpi1。
  • 128 Mb QSPI NOR Flash PY25Q128HA，走 mpi2，连接模组 PA_12–PA_17。
  • 48 MHz 主晶振（hxt48），32.768 kHz 低速晶振（PA_22/PA_23，lxt32）。
显示与触摸
  • 22P 0.5mm QSPI 屏幕 FPC（J0102），支持 SPI/DSPI/QSPI（含 DDR）和 MCU/8080 屏。
  • 设备树默认配置 chipone,co5300-qspi AMOLED（390 × 450，QSPI，50 MHz）作为示例屏。
  • 官方列出的可选屏型号：TFT-H043A28WQISTKN22（QSPI 接口）。
  • 触摸为 I2C 接口：SDA PA_33、SCL PA_30（FPC pin19/20）。
  • 背光为 FPC LEDA/LEDK，由板载升压 AW9962 驱动，PWM 控制脚 PA_01。
USB、串口与调试
  • 两个 Type-C：一个经 CH343P USB 转串口用于下载/调试；一个为原生 USB 2.0 FS。
  • 调试串口 PA_19 TXD / PA_18 RXD，Zephyr 控制台速率 1000000。
  • 原生 USB：PA_35 DP / PA_36 DM（USB-C J0114）。
  • 40P 排针另引出一路 UART：PA_27 TXD（pin8）/ PA_20 RXD（pin10）。
电源与电池
  • 供电：双 USB Type-C 或单节锂电池（电池连接器 J0100，2.0mm BTB）。
  • 线性充电 IC ETA9640P，官方文档：最大 1 A、默认 450 mA。
  • 系统电源 TLV62569 DC-DC，BL8077 ×2 LDO（VSYS → 3.3V / AVDD_3V3）。
  • 40P 排针引出 3V3（pin1/17）、5V（pin2/4）和多个 GND。
音频
  • 板载 MEMS MIC WMM7037（MIC_BIAS / MIC_ADC_IN）。
  • 模拟音频输出 AU_DAC1P/N_OUT，经 Class-D 功放 NS4150B 到喇叭连接器 J0101。
  • 功放使能 PA_10（设备树 audcodec 的 pa-power-gpios）。
扩展与板载外设
  • 40P 树莓派风格 2×20 排针（J0117），引出 LCD QSPI、SPI/SD、I2C、UART、GPIO、电源。
  • Micro SD 卡槽（J0115），SPI1：PA_24/25/28/29，检测 PA_26。
  • 板载 SK6812MINI-HS RGB LED（数据 PA_32）。
  • 用户 LED（蓝，D0104，PA_26，与 TF 检测共用网络）；充电状态绿/红 LED 由充电 IC 驱动。
  • 按键 S0100（PWKEY/HOME，PA_34）、S0101（功能键 KEY2，PA_11）、S0102（板载复位/启动键）。
  • 两片模拟开关 TS5A3157（U0110/U0111）+ 0 Ω 跳线,在 40P 排针与 22P FPC / SDIO 之间复用部分 GPIO。

后续待补充

本页已把能从官方页、V1.2.0 网表和当前设备树直接确认的引脚与资源整理完毕,引脚映射均与设备树核对一致。下列项目建议后续按实际使用补齐:

  • 实际安装的屏幕 / 触摸型号(官方示例为 TFT-H043A28WQISTKN22,设备树示例为 co5300 1.85" AMOLED),据此确定触摸 INT/RST 用 PA_31 还是 PA_09、I2C 地址与极性。
  • RGB LED SK6812(PA_32)的 Zephyr 驱动策略（led-strip / SPI / PWM bitbang）。
  • 背光 PA_01 + AW9962 的 PWM/GPIO backlight 节点。
  • 原生 USB（PA_35/PA_36）与 40P 第二路 UART（PA_27/PA_20）如需启用时的 pinctrl/节点。
  • 40P 排针 3/5/7/13/15/16 脚的 0 Ω 跳线默认贴装位置(PA 组 vs SDIO),以实物为准。
