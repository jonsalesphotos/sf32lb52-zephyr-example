## 1. 开发板说明

黄山派是基于 SiFli **SF32LB52-MOD-1** 模组打造的智能手表 / 手环原型板。

### 1.1 核心规格

| 项目     | 参数 |
|----------|------|
| 主控芯片 | **SF32LB525UC6**(Arm Cortex-M33) |
| 模组     | SF32LB52X-MOD-1-N16R8 |
| Flash    | 128 Mb(16 MB)QSPI-NOR,72 MHz,STR 模式 |
| PSRAM    | 8 MB OPI-PSRAM,144 MHz |
| 晶振     | 48 MHz + 32.768 kHz |
| 版本     | V1.0.0(内部试用) |

### 1.2 显示

- 1.85″ AMOLED,分辨率 **390 × 450**,亮度 800 cd/m²
- 接口:Quad SPI(支持 DDR),22-pin FPC
- 驱动 IC:**CO5300AF-01**(OLED)、BV6802W(电源)、**FT6146-M00**(触摸)

### 1.3 主要外设

- **音频**:MEMS 麦克风 + Class-D 功放(最高 3 W / 4 Ω 扬声器)
- **传感器**:6 轴 IMU(LSM6DS3TR-C)、磁力计(MMC5603NJ)、环境光传感器 LTR-303ALS-01
- **存储**:Micro SD(SPI 接口)
- **USB**:Type-C,板载 CH340N(下载 / 调试,RTS 触发硬件复位)
- **按键**:电源键(长按 10 s 复位)+ 功能键(HOME,PA_34)
- **马达**:板载驱动 + 外接马达焊盘,控制脚 PA_20,供电来自 `LDO3_3V3` / `VDD33_VOUT`
- **RGB LED**:BOM / 网表确认为 WS2812B-2020,控制脚 PA_32,供电来自 `LDO3_3V3` / `VDD33_VOUT`;SDK 驱动名可能沿用 SK6812MINI-HS 兼容类
- **电池**:GH-1.25 mm 接口,支持 450–500 mAh 锂电
- **充电**:BOM / 网表确认为 AW32001ECSR,本地驱动按 AW32001,最大 500 mA(默认 450 mA);早期公开资料中的 SY6103 仅作为资料差异记录
- **扩展**:30-pin 排针(USB 电源、电池、I2C、SPI、GPIO)

### 1.4 关键 GPIO / 模组信号速查

| 引脚 | 板级功能 | 本地 SDK / 备注 |
|------|----------|-----------------|
| PA_00 | LCD Reset / RSTB | QSPI 屏复位,`bsp_lcd_tp.c` 中作为 LCD reset |
| PA_01 | LCD BL PWM / 背光控制 | `LCD_BACKLIGHT_CONTROL_PIN=1`,PWM 接口默认 `pwmt1` CH4 |
| PA_02 | LCD TE | QSPI 屏同步信号 |
| PA_03 | LCD QSPI CS | 屏片选 |
| PA_04 | LCD QSPI CLK | 屏时钟 |
| PA_05 | LCD QSPI D0 | 本地 `bsp_pinmux.c` 使用 `PA05`;网页 FPC 表有 `PB_05` 写法,落代码以 SDK 为准 |
| PA_06 | LCD QSPI D1 | 同时在部分资料标 E-Paper DC / SPI DC |
| PA_07 | LCD QSPI D2 | 屏数据 |
| PA_08 | LCD QSPI D3 | 屏数据 |
| PA_09 | Touch Reset | 触摸复位,`TP_RESET` |
| PA_10 | Charger I2C SCL | 本地 SDK 使用 `I2C2_SCL`,充电芯片 AW32001 地址 0x49 |
| PA_11 | Charger I2C SDA | 本地 SDK 使用 `I2C2_SDA` |
| PA_18 | Debug UART RXD | 程序下载 / 日志串口 |
| PA_19 | Debug UART TXD | 程序下载 / 日志串口 |
| PA_20 | Vibrator PWM | 马达 / 振动 PWM；马达供电来自 `VDD33_VOUT` |
| PA_24 | SPI1 DIO / SD DO | TF 卡 SPI 信号,扩展时插 TF 卡会占用 |
| PA_25 | SPI1 DI / SD DI | TF 卡 SPI 信号 |
| PA_26 | VCC_3V3 使能 / VCC33 EN | PA_26 → ETA5055V330DS2F (U4) EN，DTS `regulator-always-on` 始终开启；不是 LED！ |
| PA_27 | SD_DET | TF 卡检测 |
| PA_28 | SPI1 CLK | TF 卡 SPI 时钟 |

| PA_31 | Sensor INT1 | IMU / 传感器中断预留 |
| PA_32 | RGB LED | `RGBLED_CONTROL_PIN=32`；LED3 VDD 来自 `VDD33_VOUT` |
| PA_33 | Touch I2C SDA | 触摸 I2C SDA,本地 SDK 走 `I2C1_SDA` |
| PA_34 | KEY1 / HOME / 长按复位 | `CONFIG_BSP_KEY1_PIN=34`;PA_34 同时经 Q1(MOSFET)反相驱动 AW32001 CHG_INT: PA_34 高→Q1 导通→CHG_INT 低→长按触发 charger shipping mode(断电) |
| PA_35 | USB DP | USB2.0 FS,也在 30P 排针引出 |
| PA_36 | USB DM | USB2.0 FS,也在 30P 排针引出 |
| PA_37 | Touch I2C SCL | 触摸 I2C SCL,本地 SDK 走 `I2C1_SCL` |
| PA_38 | VSYS_1 使能 | PA_38 → LP5240HVF (U2) EN，DTS `regulator-always-on` 始终开启，最先上电 |
| PA_39 | Sensor I2C SDA | 官方/立创表记为 Sensor I2C1_SDA;SDK sensor 示例实际把该脚复用为 `I2C3_SDA` 并用 `"i2c3"` 访问 |
| PA_40 | Sensor I2C SCL | 官方/立创表记为 Sensor I2C1_SCL;SDK sensor 示例实际把该脚复用为 `I2C3_SCL` |
| PA_41 | Touch INT | `TOUCH_IRQ_PIN=41`,触摸中断 |
| PA_42 | Audio_PA_EN / D_AMP_CTRL | 本地 `CONFIG_AW8155_GPIO_PIN=42` |
| PA_43 | KEY2 | `CONFIG_BSP_KEY2_PIN=43`,小智对话键常用 |
| PA_44 | VBUS_DET | SDK 可能命名为 `CONFIG_BSP_CHARGER_INT_PIN=44`,但网表为 VBUS 分压检测;不是 AW32001 U3.B2 INT |

### 2.2 `board.conf` 中实际启用的驱动(HCPU)

```
LCD       : CONFIG_LCD_USING_TFT_CO5300=y      → CO5300 OLED
触摸      : CONFIG_BSP_USING_TOUCHD=y
功放      : CONFIG_PA_USING_AW8155=y           → 控制脚 PA42
充电      : CONFIG_CHARGE_USING_AW32001=y      → I2C2,地址 0x49;PA44 是 VBUS_DET,非 U3.B2 INT
LED       : CONFIG_BSP_USING_LED1=y            → PIN26
按键      : KEY1=PIN34(HOME/电源)、KEY2=PIN43
总线      : I2C1/I2C2/I2C4、SPI1、UART1、ADC、PWMT1/PWMT2
注意      : 默认 board.conf 没启用 I2C3 和 SENSOR_USING_*;传感器示例工程单独打开这些配置
```

> **Zephyr DTS regulator 策略**: `vsys_en`(PA_38)和 `vcc_3v3`(PA_26)使用 `regulator-always-on` 始终开启；`gs_3v3`(PA_30)使用 `regulator-boot-off` 默认关闭，传感器 bring-up 时由应用层 `regulator_enable()` 开启。详见 [power-tree.md](power-tree.md) §5.1。

### 3.1.1 黄山派传感器 Pin 映射与 xiaozhi-sf32 对应位置

> `E:\OpenSiFli\xiaozhi-sf32` 直接使用黄山派 `sf32lb52-lchspi-ulp` 作为板级基础。该工程里的 SDK 副本位于 `E:\OpenSiFli\xiaozhi-sf32\sdk`，黄山派板级目录为 `customer/boards/sf32lb52-lchspi-ulp/` 与 `customer/boards/sf32lb52-lchspi-ulp_base/`。
>
> 立创 Sensor 文档特别说明:使用板载传感器时需要同时拉高 `PA_38` 与 `PA_30`。`PA_38` 控制 VSYS 到 VSYS_1 路径,`PA_30` 控制 Sensor Power/HR3V3;任一未使能都可能导致传感器无法通讯。

| 器件 | 作用摘要 |
|------|----------|
| LTR303 | 数字环境光传感器,用于亮度检测和自动调光,量程资料给到 0.01–88,000 lux |
| MMC56x3 | 三轴磁传感器,用于检测磁场强度和方向,可服务指南针 / 姿态辅助 |
| LSM6DS3TR-C / LSM6DSL | 6 轴 IMU,含三轴加速度计和三轴陀螺仪,支持姿态、抬腕、计步等应用 |

| 器件 | 传感器 / 外设功能 | Pin / 总线关系 | 驱动位置 | 初始化入口 / 备注 |
|------|-------------------|----------------|----------|-------------------|
| LSM6DS3TR-C / LSM6DSL 系列 | 6 轴 IMU(加速度、陀螺、计步) | `PA_39=I2C3_SDA`、`PA_40=I2C3_SCL`、`PA_38=VSYS_1 Enable`、`PA_30=Sensor Power Enable`、`PA_31=Sensor INT1` | `customer/peripherals/sensor/LSM6DSL/` | `rt_hw_lsm6dsl_init()`；示例位于 `example/rt_device/sensor/src/main.c` |
| MMC5603NJ / MMC56x3 系列 | 三轴磁力计 | 与板载传感器共用 `I2C3`: `PA_39/PA_40`，电源共用 `PA_38/PA_30` | `customer/peripherals/sensor/MMC56x3/` | `rt_hw_mmc56x3_init()`；示例位于 `example/rt_device/sensor/src/main.c` |
| 环境光传感器 / LTR303 | 环境光 ALS | 与板载传感器共用 `I2C3`: `PA_39/PA_40`，电源共用 `PA_38/PA_30` | `customer/peripherals/sensor/LTR303/` | `rt_hw_ltr303_init()`；示例位于 `example/rt_device/sensor/src/main.c` |
| 环境光传感器 / TSL2572 | 环境光 / 接近传感器备选 | 驱动默认 `TSL2572_I2C_NAME="i2c4"`，若实际接黄山派板载 Sensor I2C，需要改到 `i2c3` | `customer/peripherals/sensor/TSL2572/` | `rt_hw_tsl2572_init()`；是否使用需按实际 ALS 料号确认 |
| FT6146-M00 | 电容触摸 | `PA_33=I2C1_SDA`、`PA_37=I2C1_SCL`、`PA_41=CTP_INT`、`PA_09=TP_RESET` | `customer/peripherals/touch_panel/ft6146/` | 由触摸框架 `drv_touch.c` 注册为 `"touch"` |

黄山派传感器示例工程 `example/rt_device/sensor/` 已给出最直接的初始化方式:

```c
HAL_PIN_Set(PAD_PA40, I2C3_SCL, PIN_PULLUP, 1);
HAL_PIN_Set(PAD_PA39, I2C3_SDA, PIN_PULLUP, 1);

rt_pin_mode(GET_PIN(1, 38), PIN_MODE_OUTPUT);
rt_pin_write(GET_PIN(1, 38), PIN_HIGH);
rt_pin_mode(GET_PIN(1, 30), PIN_MODE_OUTPUT);
rt_pin_write(GET_PIN(1, 30), PIN_HIGH);
rt_thread_mdelay(50);

cfg.intf.dev_name = "i2c3";
rt_hw_ltr303_init("ltr303", &cfg);
rt_hw_mmc56x3_init("mmc56x3", &cfg);
rt_hw_lsm6dsl_init("lsm6d", &cfg);
```

`xiaozhi-sf32/app` 当前默认工程未启用 `CONFIG_SENSOR_USING_*`，也未启用 `CONFIG_BSP_USING_I2C3`；应用板级文件 `app/src/board/sf32lb52-lchspi-ulp.cc` 只在低功耗 pinmux 中处理了 `PA_30/PA_39/PA_40`。如果要在小智工程里读取 IMU / 磁力计 / 光照，需要把 `example/rt_device/sensor/project/proj.conf` 中的 `I2C3` 与三个传感器配置移入应用工程，并在板级初始化阶段保留 `PA_39/PA_40` pinmux 与 `PA_38/PA_30` 上电逻辑。

| 设备 | `rt_device_find()` 名称 | 读数 |
|------|-------------------------|------|
| LTR303 | `"li_ltr303"` | `data.light` lux |

## 6. 本地网表 × SDK × Zephyr DTS 三方交叉验证

> 本文件已同步进 Zephyr 仓库:`application/boards/sifli/sf32lb52_hspi/doc/`(该板即官方 `sf32lb52-lchspi-ulp`)。
> 三个数据源:
> - **本地网表**(最权威的硬件连线):`Netlist_黄山派核心板_SCH_V1.2_2026-06-24.tel`、`Netlist_黄山派LCD屏幕板_SCH_V1.1_2026-06-24.tel`,原理图 `PDF_立创·黄山派…`。

>

### 6.3 剩余缺口 / bring-up 风险

- **传感器电源有顺序要求**(`bsp_power.c` 上电序列):`PA_38`(VSYS,最先)→ `PA_26`(VCC_3V3)→ `PA_30`(GS_3V3),随后**强制 500 µs 延时**才能用传感器/充电 I2C。DTS 中 `vsys_en` 和 `vcc_3v3` 已设为 `regulator-always-on` 自动开启；`gs_3v3` 使用 `regulator-boot-off` 默认关闭，传感器 bring-up 时由应用层 `regulator_enable(DT_NODELABEL(gs_3v3))` 开启(需 `CONFIG_REGULATOR=y`)。
- **LCD 背光 PWM**:`PA_01` = GPTIM1_CH4(pwmt1 ch4)。DTS 无背光/PWM 节点。
- **外设节点多为 disabled**:FT6146、LSM6DSL/MMC56x3/LTR303、AW32001、WS2812B 当前均已建节点但默认 disabled,需要逐个 bring-up。
- **`LDO3_3V3` / `VDD33_VOUT`**:DTS 已按内部 regulator 建模,用于 LED3 和马达供电；它不是外部 3.3 V 输入,负载总电流需按模组 150 mA 输出预算核算。
- **马达 PWM `PA_20`**:DTS 已建 `pwm-vibrator` 占位节点但默认 disabled,避免启动误振动；启用时还需确认 PWM 频率、占空比和启动电流。
- **SD 检测 `PA_27`、原生 USB `PA_35/36`**:网表有,DTS 未建。
- LTR-303 仍用占位 `liteon,ltr329`;AW32001 仍用占位 `awinic,aw32001`(地址 0x49、总线 i2c2 已确认),需要驱动确认。

> 交叉验证用工具:本地网表解析 + BOM 抽取 + 官方 SDK 源码(`bsp_pinmux.c`/`board.conf`/驱动头) + Zephyr `uart_shell` 构建。构建验证命令:`west build -b sf32lb52_hspi application\samples\uart_shell -d build\sf32lb52_hspi_audit --pristine always`。
