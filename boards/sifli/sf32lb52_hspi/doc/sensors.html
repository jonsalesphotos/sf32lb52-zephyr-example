HSPI-SF32LB52 板载传感器与器件文档

立创·黄山派(sf32lb52_hspi)板载传感器及相关 I2C/外设器件的资料汇总:引脚、I2C 地址、Zephyr 驱动 compatible、设备树节点状态、关键规格与本地 datasheet。 引脚以 V1.2 黄山派网表(Netlist_黄山派核心板)+ BOM 为权威来源。所有节点当前在设备树中均为 disabled,后续逐个启用。

状态说明

disabled 设备树中已建节点但未启用(占位,待逐个 bring-up) 驱动可用 Zephyr 有对应 upstream 驱动 占位 compatible 暂无 upstream 驱动,compatible 为占位

当前 sf32lb52_hspi.dts 把所有传感器/外设节点设为 disabled,且总线 i2c2 也未启用。 启用某个器件时:把对应节点(必要时还有所在 I2C 总线)改为 status = "okay",并在应用侧打开对应 CONFIG_*(见下方"启用步骤")。

传感器(i2c3)

LSM6DS3TR-C 驱动可用 disabled 六轴惯性单元 · U13 · ST

功能
3D 加速度 + 3D 陀螺仪(iNEMO)。量程 加速度 ±2/±4/±8/±16 g,角速度 ±125/±250/±500/±1000/±2000 dps。
I2C 地址
0x6A(SA0 接地;SA0 拉高则 0x6B)。WHO_AM_I = 0x6A。
引脚
SDA PA_39 / SCL PA_40;中断 INT1 PA_31(高有效)。
Zephyr
compatible st,lsm6dsl(LSM6DS3TR-C 与 LSM6DSL 寄存器兼容、WHO_AM_I 同为 0x6A)。驱动 CONFIG_LSM6DSL,触发 CONFIG_LSM6DSL_TRIGGER。
节点
lsm6dsl@6a
Datasheet
LSM6DS3TR-C_datasheet_ST.pdf(ST)

MMC5603NJ 驱动可用 disabled 三轴地磁(磁力计) · U14 · MEMSIC

功能
单片三轴 AMR 磁传感器。量程 ±30 Gauss,20-bit,噪声 ~2 mG,航向精度 ~1°。支持 I2C 快速模式 / I3C。
I2C 地址
0x30(固定)。
引脚
SDA PA_39 / SCL PA_40(无中断脚引出)。
Zephyr
compatible memsic,mmc56x3(覆盖 MMC5603 系列)。驱动 CONFIG_MMC56X3。
节点
mmc56x3@30
Datasheet
MMC5603NJ_datasheet_MEMSIC.pdf(MEMSIC Rev.B)

LTR-303ALS-01 占位 compatible disabled 环境光传感器 · U16 · Lite-On

功能
数字环境光(ALS)。检测范围 0.01–64k lux,16-bit ADC,6 档增益(1/2/4/8/48/96×),可设积分/测量时间。
I2C 地址
0x29(固定)。
引脚
SDA PA_39 / SCL PA_40。
Zephyr
无专用 upstream 驱动。liteon,ltr329 寄存器相近但未经验证,当前用作占位 compatible,节点 disabled。后续需确认或自写驱动。
节点
ltr303@29
Datasheet
LTR-303ALS-01_datasheet_LiteOn.pdf(Lite-On v1.1)

AW32001(E) 占位 compatible disabled 线性充电 IC · U3 · Awinic

功能
Li-ion/Li-Po 线性充电,I2C 可编程充电参数,输入 OVP 至 28V,Vchg 精度 ±0.5%。

启用步骤(逐个 bring-up)

以加速度计为例,启用某器件的通用流程:

# 1) 设备树:把器件节点(必要时连同所在 I2C 总线)改为 okay
#    例如在应用 overlay 里:
&i2c3   { status = "okay"; };          # 传感器总线(本就 okay)
&lsm6dsl { status = "okay"; };          # 加速度计节点

# 2) 应用 prj.conf:打开子系统与驱动
CONFIG_I2C=y
CONFIG_REGULATOR=y          # 让 GS_3V3 传感器电源轨在 boot 时上电(PA_30)
CONFIG_SENSOR=y
CONFIG_LSM6DSL=y
# CONFIG_LSM6DSL_TRIGGER_GLOBAL_THREAD=y   # 需要 INT1(PA_31)中断时

# 充电器(i2c2)启用前还需先解决 PA_10(音频PA使能)/ PA_11(key2)复用
总线 i2c2 默认 disabled;其 pinctrl(PA_10/PA_11)在 disabled 状态下不会下发,因此与音频 PA_10、按键 PA_11 不产生运行期冲突 —— 但真正启用充电器前必须先在硬件层面/设备树层面理清这两个复用。
