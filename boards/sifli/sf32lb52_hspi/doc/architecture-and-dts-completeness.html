0. 快速结论(TL;DR)

  1. 工程架构:标准 Zephyr west workspace,以 application/ 为 manifest 仓库,拉取 OpenSiFli 的 zephyr-downstream fork 与 hal_sifli 模块。黄山派板级 sf32lb52_hspi 放在应用仓库内(out-of-tree board),继承自 fork 内的 sf32lb52_devkit_lcd。
  2. DTS 器件覆盖度:板上所有主要器件均已出现在 DTS 中——屏幕、触摸、三颗传感器、充电 IC、三路板级稳压器 + 两路片内 LDO、按键、RGB 灯、振动马达、音频功放使能全部有节点;其中屏幕/电源/音频/存储/蓝牙/控制台等 13 类已启用,触摸/传感器/充电/RGB/马达 7 个节点 disabled 占位(策略合理,待 bring-up)。
  3. 缺口状态(2026-07-02 已补全,见 §2.6):电池 GPADC 通道 7 ✔ TF 卡节点 + PA_27 ✔ PA_44 VBUS 检测 ✔ PA_01 网表证实 NC(无需建模);剩余:USB(SoC 级) SDIO(SoC 级) AW32001 驱动(绑定已补)
  4. 与 SDK 的差异多数是本工程更正确(PA_26 非 LED、PA_44 非充电 INT,均经网表核实)。PA_01 矛盾已定论:核心板网 PA_01_LCD_BL_PWM 经 22P FPC pin 2 映射到屏幕板 FPC pin 21(映射关系"屏侧 = 23 − 核心板侧",14 个信号逐一吻合),屏幕板 V1.1 网表中该脚 NC 悬空(屏幕板为纯无源转接板,电源 IC 在屏排线模组内)。SDK 上电拉高 PA_01 在本屏版本上是无操作,DTS 不建模是正确的。

1.4 hal_sifli 模块

  • cmsis/sf32lb52x/:寄存器定义、启动文件、LCPU 固件安装/初始化(lcpu_install.c、bf0_lcpu_init.c)。
  • hal/:SiFli BF0 HAL(与 SiFli-SDK 同源);Zephyr 驱动多为「薄封装 + HAL」结构。
  • ll/ 低层驱动;ipc_queue/ HCPU↔LCPU 邮箱队列(蓝牙 HCI 运输层)。

1.5 双核与蓝牙

SF32LB52x = HCPU(Cortex-M33,跑 Zephyr)+ LCPU(蓝牙控制器)。链路:bt 子系统 → hci_sf32lb.c → ipc_queue/mailbox → LCPU 固件。板级 zephyr,bt-hci = &mailbox 即此。

2.3 引脚级对照结论(Zephyr pinctrl vs SDK bsp_pinmux)

  • PSRAM(MPI1):Zephyr mpi1_default 与 SDK board_pinmux_psram_func1_2_4(1)(APS 64P XCELLA,8 MB OPI)逐脚一致。SDK 按芯片 PID 运行时选表,Zephyr 静态声明——对 525UC6 正确。
  • NOR(MPI2):SDK 显式配置 PA_12~17;Zephyr 不配置(无 pinctrl-0),依赖引导 ROM 在 XIP 启动前配好。可用,但属"隐式契约"。
  • UART1/I2C1/I2C2/I2C3/SPI1/LCDC/LXT32:两边逐脚一致。
  • SDK 有而 Zephyr 无:仅剩 PA_35/36(USB,受限于 SoC 级无 usbd 节点)。原差异项已于 2026-07-02 处理:PA_01 网表证实 NC、PA_27 已作 cd-gpios、PA_44 已入 zephyr,user。另注意 PA_29 有意从 SPI1_CS 复用改为 GPIO 片选(cs-gpios),与 SDK 硬件 CS 方案不同,原因见 §2.6。
  • SDK board.conf 的 BSP_USING_I2C4:基板无 I2C4 pinmux,疑为遗留配置,不构成缺口。

3. 学习路径建议(从零上手本工程)

  1. 跑通闭环:west build -b sf32lb52_hspi application/samples/uart_shell → west flash → 1 Mbps 串口进 shell,玩 device list、kernel threads、bt 命令(验证 LCPU 蓝牙链路)。
  2. 读板级 DTS(配合 dts-reference.md 逐行解析):sf32lb52_hspi.dts → sf32lb52x.dtsi → 各绑定 yaml,理解 chosen/regulator/pinctrl 三大块。
  3. 第一个 bring-up 实验(推荐传感器):应用里 regulator_enable(gs_3v3) + 延时,overlay 把 lsm6dsl 置 okay,用 sensor shell 读数——完整体验"电源时序 + overlay + 驱动绑定"链路。
  4. 显示/触摸:显示已 okay,可直接跑 lvgl(模块已拉取);触摸置 okay 验证 input 子系统。(PA_01 已证实 NC,屏幕排障时无需再查该脚。)
  5. 功能启用练手:DTS 侧缺口已补全,应用层直接启用即可——TF 卡(CONFIG_DISK_DRIVER_SDMMC + fatfs 挂载 "SD")、电池电压(CONFIG_ADC,读 zephyr,user 的 "vbat" 通道)、VBUS 检测(GPIO 读 vbus-det-gpios)。进阶题目:AW32001 充电驱动、SoC 级 USB/SDIO 支持。
  6. 架构深潜:hci_sf32lb.c + ipc_queue(双核)、memc_sf32lb_mpi_opi_psram.c(存储)、SDK bsp_power.c(低功耗流程,Zephyr 侧 PM 尚未对齐——进阶方向)。
