4 · 对应的 SMF 机制

  • 分层 + 初始转换 — ACTIVE 是 IDLE/HEATING/COOLING 的父态,并把 initial 指向 IDLE。 需要 CONFIG_SMF_ANCESTOR_SUPPORT=y 与 CONFIG_SMF_INITIAL_TRANSITION=y。
  • run 向上冒泡(v0.2.0 新接口) — 叶子 run 对自己不处理的事件返回 SMF_EVENT_PROPAGATE,框架接着调用父 active_run;处理掉则返回 SMF_EVENT_HANDLED(或调用 smf_set_state())以停止冒泡。 旧版的 smf_set_handled() 在本版本已由返回值取代。
  • LCA / 兄弟态优化 — IDLE↔HEATING↔COOLING 的最近公共祖先是 ACTIVE,所以这些跳转不会重跑 ACTIVE 的 entry/exit; 而进/出 FAULT(父=NULL)会经过 ACTIVE 的 exit/entry。
  • 终止 — smf_set_terminate(ctx, val) 的 val 必须非 0, 因为 smf_run_state() 会把它作为返回值,主循环以“非 0 即停止”判断。传 0 会导致空转。
  • 事件驱动 — main() 把脚本事件 k_event_post 给状态机线程, 线程 k_event_wait 取事件后调用 smf_run_state()。这是真实工程里 中断 / 其他线程投递事件、状态机线程消费的标准结构(需 CONFIG_EVENTS=y)。
